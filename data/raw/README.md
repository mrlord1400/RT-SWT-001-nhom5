## 3. Cấu trúc dữ liệu (Data structure)

Dataset gồm **22 file `.txt`**, mỗi file tương ứng với **một dự án/sản phẩm phần mềm khác nhau** (lấy từ các nguồn như GitHub, trang dự án mã nguồn mở, hệ thống quản lý yêu cầu công khai...). Mỗi dòng trong file là **một user story** duy nhất, không có header, không có cột phân tách — toàn bộ là văn bản thuần (plain text), mỗi user story kết thúc bằng dấu xuống dòng.

### Định dạng user story (template chung)
Phần lớn các dòng theo cấu trúc câu chuyện người dùng kinh điển:

```
As a <role>, I want <feature/goal> [, so that <reason/benefit>].
```

Trong đó:
- **`role`**: vai trò/persona của người dùng (ví dụ: *Developer*, *UI designer*, *Agency user*, *faculty member*...)
- **`feature/goal`**: chức năng hoặc mục tiêu mong muốn
- **`reason/benefit`**: lý do/lợi ích (phần này **không bắt buộc** — nhiều user story trong dataset không có mệnh đề "so that")

## 4. Cách dataset được dùng trong đồ án này

- Thư mục `data/raw/` chỉ chứa dữ liệu **gốc, chưa qua xử lý** — **không sửa/ghi đè** lên các file `.txt` này.
- Mọi bước làm sạch (loại bỏ `\t`, `\r`, dòng trống), gán nhãn (role/feature/reason), hoặc chuyển sang `.csv`/`.json` sẽ được thực hiện ở các bước xử lý tiếp theo (ví dụ trong `data/processed/`), không nằm trong phạm vi của file README này.
