# Inclusion and Exclusion Criteria

## Inclusion Criteria

* **IC1 (Topic):** Uses LLMs to generate BDD/Gherkin scenarios or step definitions.  
* **IC2 (Evaluation):** Contains actual quantitative data comparing the LLM's results against a manual/human baseline.  
* **IC3 (Timeline):** Published from 2020 onwards.  
* **IC4 (Quality & Access):** Peer-reviewed (or a reputable pre-print) and written entirely in English.  

## Exclusion Criteria

* **EC1 (Wrong Tech):** Uses older, traditional machine learning or rule-based methods instead of modern LLMs.  
* **EC2 (Wrong Scope):** Generates standard unit tests instead of BDD, or bases tests on heavy code/diagrams instead of natural language User Stories.  
* **EC3 (No Data / Too Short):** It is an opinion piece, survey, or a paper shorter than 4 pages that lacks concrete test results.  
* **EC4 (Inaccessible / Duplicate):** It is a duplicate study, or you cannot access the full text.
